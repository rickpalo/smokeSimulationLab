Overview

SmokeSimLab automates the process of exploring smoke simulation parameters in Blender. Instead of manually changing settings, baking, and rendering one at a time, you define parameter ranges in a panel, click Export Batch, and a Windows batch file runs every combination automatically.

For each job the addon: bakes the Mantaflow fluid simulation, renders an MP4 playblast animation, renders a final quality PNG still frame, updates in-scene text objects with the current parameter values, and logs everything to a CSV file.

Why batch files? Blender 4.5 crashes when bpy.ops.fluid.bake_all() is called from inside another Python operator or timer. Launching separate Blender processes avoids this entirely — each process owns its own memory and event loop.

Installation

Download SmokeSimLab.zip from the Releases page.
In Blender open Edit → Preferences → Add-ons.
Click Install and select SmokeSimLab.zip.
Enable SmokeSimLab in the add-on list.
Press N in the 3D Viewport and open the SmokeLab tab.
Both files must be installed together. The zip contains __init__.py and smoke_worker.py. If you manually copy only one file the Export Batch button will show an error.

Quick Start

Set up a smoke simulation scene with a Mantaflow fluid domain object.
Open the SmokeLab tab in the N-panel.
Set Domain Object to your fluid domain.
Set Output to a folder for results.
Configure parameter defaults and any ranges to sweep.
Save your .blend file — the batch launcher needs its absolute path.
Click Export Batch.
Double-click run_smoke_batch.bat in Windows Explorer.

Parameter Reference

Resolution

The number of voxels along the longest side of the fluid domain. Blender's default is 32. Higher values produce more detail but bake times increase roughly as the cube of the resolution.

Gas Parameters

All three gas parameters live in a single collapsible "Gas Parameters" section.

ParameterBlender AttributeDefaultRangeDescription
Vorticity d.vorticity 1.0 0 – ∞ Adds turbulent swirling detail. 0 = smooth rising column, higher = chaotic swirling smoke.
Buoyancy Density d.alpha 1.0 –5 – 5 Buoyant force from smoke density. Positive = smoke rises, negative = sinks.
Buoyancy Heat d.beta 1.0 –5 – 5 Buoyant force from temperature. Works with flow object Initial Temperature setting.

Dissolve

When enabled, smoke fades out over time rather than accumulating indefinitely. The enable checkbox appears in the section header.

OptionDescription
Dissolve SpeedNumber of frames until smoke fully dissolves. Blender default is 5.
Slow DissolveUse logarithmic fade instead of linear — smoke lingers longer then fades quickly.

Noise

Adds a high-resolution turbulent detail layer on top of the base simulation. The enable checkbox appears in the section header.

ParameterBlender AttributeDefaultDescription
Scale d.noise_scale 2 Upres factor — how many times finer the noise grid is vs the base grid.
Strength d.noise_strength 1.0 How strongly the noise detail affects the visual result.
Position Scale d.noise_pos_scale 1.0 Spatial frequency of the noise pattern. Higher = finer, more detailed noise.

Input Modes

Every parameter supports three input modes, selectable with the Range / List toggles:

ModeHow to activateBehaviour
Default only Neither Range nor List enabled The default value is used for every job. No iteration on this parameter.
Range Enable the Range toggle Values are generated from Begin to End in increments of Step.
List Enable the List toggle Specific values entered by the user via the + / − buttons.
Tip: Range and List are mutually exclusive — enabling one automatically disables the other.

Iteration Modes

Limited Combinations

  • Vary one parameter at a time
  • All others stay at their default value
  • Total jobs = sum of range lengths
  • Best for initial exploration

All Combinations

  • Full Cartesian product of all ranges
  • Total jobs = product of range lengths
  • Can produce very many jobs quickly
  • Best for fine-tuning after exploration

Example

Suppose you define: Vorticity range [0.5, 1.0, 1.5] and Noise Strength range [0.5, 1.0, 1.5]. All other parameters at default.

ModeJobsDescription
Limited 6 3 vorticity jobs + 3 noise strength jobs
All 9 3 × 3 = every combination

With five parameters each having a 5-step range, Limited = 25 jobs vs All = 3,125 jobs. Limited Combinations is strongly recommended for initial parameter exploration.

Render Engine Options

ModeSpeedQualityRequirements
Cycles GPU (default) Medium Path-traced Works in --background mode. Uses OptiX → CUDA → CPU fallback.
EEVEE Fast Rasterised Requires a visible Blender window (windowed mode). Cannot run headless.
Playblast samples: The playblast uses 32 Cycles samples by default for speed. Edit smoke_worker.py and change scene.cycles.samples = 32 to adjust quality vs speed. Re-export after editing the worker.

Text Objects

SmokeSimLab can update FONT (text) objects in your scene with the current job's parameter values before each render, so the values appear in the rendered output. This is useful for comparison sheets.

Expand the Text Objects section in the panel and enter the exact name of each FONT object in your scene. Objects not found are silently skipped.

FieldDefault NameExample Value
ResolutionResolution_TextRes: 64
NoiseNoise_TextNoise: U-2 | St-1.0 | Scale-1.0
DissolveDissolve_TextDissolve: Time: 50 | Slow-No
Bake TimeTime_TextBake: 5 min 23 sec
Tip: Text objects are updated in the background Blender process. The changes are written to the scene in memory and appear in the rendered frames, but are not saved back to your original .blend file.

Troubleshooting

Export Batch shows "smoke_worker.py not found"

Both files must be installed together. Re-install the addon using the zip file. Do not install __init__.py and smoke_worker.py separately.

Batch crashes immediately with EXCEPTION_ACCESS_VIOLATION

Some installed Blender addons crash when Blender loads in background mode. The --factory-startup flag prevents most user addons from loading. If this doesn't help, check jobs/job_0000.log — the last line before the crash usually identifies the problematic addon.

Cache files found: 0 after bake

The bake completed but wrote nothing to disk. Common causes:

  • Cache directory path contains special characters or spaces in unusual positions
  • The domain object's Fluid modifier is set to a type other than Domain
  • Output path is on a network drive that is slow to flush

Check the full log for error messages before the "Cache files found: 0" line.

Playblast takes a very long time

The Cycles sample count defaults to 32. For faster previews at the cost of quality, edit smoke_worker.py in your output folder and reduce scene.cycles.samples = 32 to 8 or 16. Re-export the batch after changing the worker (Export Batch overwrites the worker copy).

Physics panel resolution is greyed out after batch

The domain is pointing at a job cache directory which has existing files, so Blender locks the parameters. In Physics Properties → Fluid, click Free All, then point the cache directory back to your working folder.

The job count seems wrong

In Limited Combinations mode the count is the sum of range lengths (not the product). In All Combinations mode it is the product. The panel shows the exact count above the Export button. Also check for the floating-point range off-by-one: a range of 0.2 to 1.0 in steps of 0.2 correctly produces 5 values (the epsilon fix handles this).

EEVEE renders crash or produce blank frames

EEVEE requires an OpenGL context which is not available in --background mode. Switch to Cycles GPU in the Render Engine selector, or if you specifically need EEVEE, switch to EEVEE mode which opens a small visible Blender window per job.
