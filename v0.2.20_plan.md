## A. Bug Fixes in `__init__.py`

### A1. `generate_jobs_all` iterate_both undefined properties (CRITICAL)
**File:** `scripts/SmokeSimLab/__init__.py` lines ~506–525  
**Bug:** When `iterate_dissolve_both=True` or `iterate_noise_both=True`, the "off" state
appended to `dissolve_states`/`noise_states` references `s.dissolve_speed`, `s.noise_upres`,
`s.noise_strength`, `s.noise_spatial_scale` — none of which exist as SmokeSettings
properties. Only `_begin` variants are registered. This raises `AttributeError` in Blender
whenever a user enables either "Iterate Both On and Off" checkbox in All Combinations mode.

**Fix:**
```python
# Before (line ~506):
dissolve_states.append((False, s.slow_dissolve, [s.dissolve_speed]))
# After:
dissolve_states.append((False, s.slow_dissolve, [expand_param(s, "dissolve_speed")[0]]))

# Before (lines ~517–520):
noise_states.append((False,
                      [s.noise_upres],
                      [s.noise_strength],
                      [s.noise_spatial_scale]))
# After:
noise_states.append((False,
                      [expand_param(s, "noise_upres")[0]],
                      [expand_param(s, "noise_strength")[0]],
                      [expand_param(s, "noise_spatial_scale")[0]]))
```
